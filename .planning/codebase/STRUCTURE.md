## Workspace Type

Bun monorepo with Turborepo build orchestration. Root `package.json` defines workspaces across 6 top-level directories.

---

## Top-Level Layout

```
acovado.club/
├── apps/               # Deployable applications
├── clients/            # Client libraries / thin wrappers
├── config/             # Shared configuration packages
├── infra/              # Infrastructure definitions (Docker, compose)
├── modules/            # Core business logic (shared libraries)
├── tests/              # Test suites (e2e, simulation)
├── .changeset/         # Changeset versioning config
├── .drone.yml          # CI/CD pipeline (Drone CI)
├── .husky/             # Git hooks (pre-commit, commit-msg)
├── .lintstagedrc       # Lint-staged config
├── .commitlintrc       # Commit message linting
├── .czrc               # Commitizen config
├── biome.json          # Linter/formatter config (Biome)
├── turbo.json          # Turborepo pipeline config
├── package.json        # Root workspace definition
└── bun.lockb           # Bun lockfile
```

## Workspace Directories

### `modules/` — Core Library Modules

Shared business logic imported across apps. Each module is an independent package with its own `package.json`, `src/`, `dist/`, and `tsconfig.json`.

| Module | Purpose |
|--------|---------|
| `modules/db/` | PostgreSQL client (Drizzle ORM), schema, migrations |
| `modules/events/` | RabbitMQ producer/consumer abstractions |
| `modules/graph-db/` | FalkorDB (graph DB) client via Redis protocol |
| `modules/ids/` | ID generation utilities |
| `modules/inference/` | LLM inference client (Ollama wrapper, model-agnostic) |
| `modules/logger/` | Structured logging |
| `modules/reddit-client/` | Reddit API HTTP client |
| `modules/tracing/` | OpenTelemetry tracing SDK (OTLP export) |
| `modules/types/` | Shared TypeScript types |

**Typical module structure:**
```
modules/<name>/
├── src/
│   └── index.ts        # Main entry point
├── dist/               # Compiled output
├── package.json
└── tsconfig.json
```

## Key File Locations

| File | Location | Purpose |
|------|----------|---------|
| DB schema | `modules/db/src/` | Drizzle schema definitions |
| DB migrations | `modules/db/drizzle.config.ts` + migration files | Schema migrations |
| RabbitMQ producer | `modules/events/src/` | Message publishing |
| RabbitMQ consumer | `modules/events/src/` | Message consumption |
| Reddit HTTP client | `modules/reddit-client/src/index.ts` | Reddit API calls |
| Graph DB client | `modules/graph-db/src/client.ts` | FalkorDB queries |
| Tracing setup | `modules/tracing/` | OTLP trace export |
| CI pipeline | `.drone.yml` | Drone CI build/deploy |
| Docker Compose | `infra/*/` | Per-service compose files |

## Naming Conventions

- **Packages:** `@repo/<name>` or `@config/<name>` (kebab-case)
- **Modules:** All lowercase kebab-case directory names
- **Source files:** `index.ts` as main entry; kebab-case for other files
- **Dist output:** `dist/` adjacent to `src/`
- **Config packages:** Prefixed with `@config/`
