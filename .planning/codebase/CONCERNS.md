### Details

**Missing FK constraints on `stories`** — The `stories` table lacks foreign key constraints that enforce referential integrity. Orphaned records can accumulate silently.

**Raw SQL in migrations** — Several migration files use raw SQL strings instead of typed query builders, making them harder to audit and prone to typos.

**Duplicate annotation keys** — Annotation objects allow duplicate keys to be set, with the last write winning silently. No deduplication or validation at write time.

## Known Bugs

### BUG-01: Wrong ticker symbol in seed script
- **File:** seed script
- **Issue:** Uses `GOOGLE` instead of `GOOGL` — causes silent data loss when symbols are looked up downstream
- **Impact:** High — wrong symbol never matches real market data

### BUG-02: Unawaited `producer.send()` in simulation
- **File:** simulation/producer
- **Issue:** `producer.send()` called without `await`, so publish errors are swallowed
- **Impact:** Medium — message loss goes undetected in simulation runs

## Security

### SEC-01: Unauthenticated Reddit API, no retry on 429
- Reddit API calls are unauthenticated (anonymous rate limit: 60 req/min)
- No retry/backoff logic on 429 responses — pipeline stalls silently

### SEC-02: Cypher parameter injection risk
- **File:** Neo4j query layer
- **Issue:** Some Cypher queries use string interpolation instead of parameterized queries
- **Risk:** Injection if any user-controlled or external data reaches query construction

### SEC-03: RabbitMQ credentials in logs
- **File:** RabbitMQ connection setup
- **Issue:** Connection config (including credentials) not scrubbed before logging
- **Risk:** Credentials visible in log files/output streams

## Performance

### PERF-01: Infinite nack-requeue loop on poison messages
- **File:** consumer layer
- **Issue:** Failed messages are nacked and requeued unconditionally — no dead-letter queue or max-retry counter
- **Impact:** One bad message can loop forever and starve the queue

### PERF-02: Unawaited async spans in DB query logger
- **File:** DB query logger
- **Issue:** Async telemetry spans not awaited — spans may be dropped under load

### PERF-03: `prefetch(1)` bottleneck
- **File:** RabbitMQ consumer config
- **Issue:** Consumer prefetch set to 1, meaning only one message processed at a time per consumer
- **Impact:** Throughput ceiling hit quickly under any real load

## Fragile Areas

### FRAGILE-01: No RabbitMQ reconnection logic
- **File:** RabbitMQ connection module
- **Issue:** If the broker connection drops, the consumer/producer silently stops working
- **Fix needed:** Implement reconnect with exponential backoff

### FRAGILE-02: Unguarded `JSON.parse` on consumer messages
- **File:** consumer message handler
- **Issue:** `JSON.parse()` called without try/catch — malformed messages crash the consumer process

### FRAGILE-03: `__dirname` CommonJS assumption in `migrate.ts`
- **File:** `migrate.ts`
- **Issue:** Uses `__dirname` which is a CommonJS global — will break if the file is compiled as ESM
